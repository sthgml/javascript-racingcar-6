## 999. 소감
 지난 주차에는 구현을 완성하고나서 지원 플랫폼에 예제 테스트를 실행해보았는데 실행조차 되지 않아서 적잖은 충격을 받았었다. 그래서 이번에는 구현 완성을 하자마자 예제 테스트를 실행해 확인하고 테스트코드를 짜기 시작했다. 다행히 실행이 잘 되었다. index.js를 만들지 않았던게 화근이었던 것 같다. 

### ✏️ 지난 주차 회고
이번 주차에는 지난 주차에 남았던 아쉬움들을 해소할 수 있는 시간이었다. 기능 목록을 작성하기 전, 제공된 피드백을 꼼꼼히 확인하여 회고록을 먼저 작성했다. 어떤 미션과 요구사항이 있었고, 내가 해당 요구사항을 얼마나 만족했는지를 신호등 이모지로 표시했다. 그 중 내가 만족하지 못한 사항들을 이번 코드를 짜면서 염두에 둘 사항들로 따로 빼서 확인했다.

### ⚾️ 객체지향 프로그래밍
 이 점에 있어서는 공통 피드백 영상이 큰 도움이 되었다. 객체와 함수를 어떻게 나누는 건지 이해가 되었다. 막연하게 기능 마다 쪼갠다 생각해서 마구잡이로 쪼갰었는데, 사람과 그 사람의 역할로 비유를 하니(ex. 심판-판단하기) 코드를 짤 때에도
 ```const result = 심판.판단하기(swing);```  
이런식으로 반영이 되어서 코드만 보고도 'result 변수에 저장된 것은 심판이 스윙을 판단한 결과구나!' 를 단번에 이해할 수 있다는 것이 객체지향 프로그래밍의 의의인 것을 깨달았다. 

### 📝 기능 목록 
 이번 프리코스를 참여하면서 처음 경험 해보는 것들이 많다. 비전공자인데다가 본격적으로 공부하기 시작한지 6개월도 채 되지 않은 나로서는 이렇게 체계적으로 프로젝트를 수행해본 적이 없기 때문이다. 그 중 하나가 이 기능 목록인데 야구게임 때는 막막하기도 했고, 두루뭉술하게 ball, strike를 true, false로 반환해주면 될까? 생각했지만 막상 직접 짜보니 전혀 그렇게 구현되지 않았기 때문이다. 

 객체지향을 이해하고 나니 어떻게 함수를 나눌지 금방 방향이 잡혀 기능 목록을 수월하게 작성했고, 큰 수정 없이 그대로 그 순서에 맞춰 순조롭게 구현을 해나갈 수 있었다. 원래는 이렇게 계획을 미리 짜놓고 수행하는 스타일이 아닌데 방향과 할 일을 정해놓으니까 시간 분배를 어떻게 하면 될지, 감이 잡히고 불안하지 않아서 효율적으로 과제를 수행할 수 있었다.
